Ulriken Consulting Firebase Workshop

Velkommen til Firebase innføring. Vi skal sette opp en Firebase, og lage software som henter og skriver data i denne, uten rest api, dvs ordentlig "Backend as a Service"

Oppgave 1: Opprette databasen

Før vi kan bengynne er må vi sette opp firebase.

  • Logg på google med en gmail konto. Andre kontoer nytter ikke.
  • Gå til Firebase.com
  • Opprett et nytt firebase prosjekt (ikke kryss av for google analytics)
  • Det kommer et snurrehule frem til prosjektet er ferdig opprettet, trykk continue
  • Sett opp autentisering (bruker/passord), under Sign-in-method, velg email/password og anonymous
  • Opprett database (Cloude Firestore), klikk på Create Database, velg start in test mode og lokasjon eur3
  • Du får snurrehjul mens databasen opprettes, og du ser at du kan klikke på create collection
  • Opprett en Web applikasjon, gå til project setting og se på ikonene nede for web, ios+++
  • Klikk på Web ikonet og registrer en app (f.eks: FbFag01)
  • Nå får du opp javascript kode som kan kopieres inn i prosjektet, kopier kun fireBaseConfig.
  • Lim den inn i index.js.
  • Live Server er en plugin som kan brukes for å server html koden, stå på index og klikk på ikonet nede til høyre.

Da skal vi være klar til å begynne kodingen

Oppgave 2a: Opprette bruker og pålogging

Det ligger et skjermbilde med knapper som oppretter en bruker, og logger på denne. Det er ikke noen input felter for brukernavn (dvs epostadresse) og passord. Klikk på opprett bruker og sjekk i firebase konsollet at brukeren dukker opp, og at du kan logge på. Vi trenger ikke lage skjermbilder for brukerid og passord ennå...

Oppgave 2b: Opprette data (seeding database)

Filen seedData.html inneholder rutiner for initialisering av databasen Der er tre knapper for hver av tabellene. Det er laget JSON for initielle data for hver av tabellene. Kjør seed kunder ved å klikke på knappen. Se at data kommer inn i tabellen.

Da kan du fullføre koden for å seede de to andre tabellene, sjekk at data havner der de skal.

Kundebildene

I systemet ligger det to kundebilder, et som lister alle kundene, og et for å registrere/endre. Dette kan du se på hvordan man leser og skriver til databasen.

Oppgave 3: Produkt bildene

Se på filene to filer "produkt.html" og "produkter.html". Du skal fullføre disse etter samme lest som kundebildene.
